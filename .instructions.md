# Agent notes (repo conventions)

## Testing (required for changes)

- Run tests in **release** mode before considering proof-layer changes complete.
  - Workspace root: `cargo test -p polymesh-dart-bp --release`
  - Or inside `dart-bp/`: `cargo test --release`

Rationale: Some tests try to benchmark as well so they take long.

## Context (where to look first)

- Start with `DART_TXN_FLOWS.md` (repo-root) for a high-level settlement flow + proof-layer map.
- For deeper protocol/math details, consult the `dart-bp/docs/*.md` files (e.g., `dart-bp/docs/4.md`, `dart-bp/docs/5.md`). `dart-bp/docs/1.md` contains the system model.
