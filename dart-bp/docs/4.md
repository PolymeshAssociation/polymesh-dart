# Settlement

A settlement can have 1 or more **Leg**s as each Leg represents one asset transfer. Say Alice wants to send 10 units of asset id $at$ to Bob, then such a settlement will have 1 leg with Alice as sender and Bob as receiver.
If Alice wants to swap asset 10 units of $at$ with Bob for 20 units of $at'$, then such a settlement will have 2 legs with Alice acting as sender in one leg and receiver in other and vice versa for Bob.

Settlements might be created by sender, receiver or a third party. To keep the amount, and identities of parties confidential, the settlement creator encrypts these details including asset-id so that the chain does not even learn which asset is being traded. However, the chain knows how many legs the settlement has. The encryption is done for all parties to that leg which include sender, receiver and all auditors and mediators for that asset.
Following describes the creation of a leg, its encryption, decryption and the proof that the leg is correctly generated.

There are some options a creator has: It can choose to reveal the asset-id to the verifier (chain) making the proofs if the use-case permits, much proofs more efficient. 
It can also decide if sender and receiver learn each other's public key. Finally, it can choose to include additional auditors and mediators than already associated with the asset 
but these auditors and mediators keys become public and not hidden like the auditors and mediators keys associated with the asset. 

Note: Following is the proof for 1 leg only. A settlement is a collection of legs and thus collection of following proofs except the curve tree membership proof is batched.


## Leg

A leg has:

1. $EK_s$ - Encryption public key of sender
2. $EK_r$ - Encryption public key of receiver
3. $v$ - Amount of units being transferred
4. $at$ - Asset-id being transferred
5. $enc\_keys$ - List of encryption keys stored with the asset (hidden from verifier unless asset-id is revealed). These keys belong to auditors and mediators who share encryption keys.
6. $med\_keys$ - List of pairs $(idx, MK)$ where $idx$ is the index in $enc\_keys$ and $MK$ is the mediator's affirmation public key (hidden from verifier unless asset-id is revealed)
7. $public\_enc\_keys$ - List of encryption keys provided by leg creator (always revealed to verifier)
8. $public\_med\_keys$ - List of pairs $(idx, MK)$ where $idx$ is the index in $public\_enc\_keys$ and $MK$ is the mediator's affirmation public key (always revealed to verifier)

### Encryption
The settlement creator wants to ensure that the same leg encryption is given to all parties and thus uses twisted Elgamal.

Note that an encryption key can be shared between an auditor and a mediator (see system model in 1.md). The issuer distributes the corresponding encryption secret key using key-distribution-proof to all such parties before they start auditing/mediating. The $Eph_{k_i}$ in the leg is for a single $EK_i$, so if a mediator shares the same $EK$ as an auditor at index $idx$, they use $Eph_{k_{idx}}$ to decrypt.

1. Creator picks 3 (or 4) independent random scalars $r_1, r_2, r_3 \in \mathbb{Z}_p$ (and optionally $r_4$ if asset-id is encrypted) and creates ciphertexts:
   $$CT_s = r_1.G_{Enc} + r_1^{-1}.EK_s$$
   $$CT_r = r_2.G_{Enc} + r_2^{-1}.EK_r$$
   $$CT_v = r_3.G_{Enc} + v.H$$
   $$CT_{at} = r_4.G_{Enc} + at.H \quad \text{(only if asset-id is not revealed)}$$

   The ciphertexts use twisted Elgamal: $CT_s$ encrypts the sender's encryption key using $(r_1, r_1^{-1})$. Sender decrypts by computing $CT_s - r_1.G_{Enc} = r_1^{-1}.EK_s$, then multiplies by $r_1$ to get $EK_s$.

2. For each asset encryption key $EK_i$ (auditors, and mediators sharing a key), create ephemeral keys to allow them to decrypt $CT_s, CT_r, CT_v$ (and optionally $CT_{at} if asset-id is encrypted)$:
   $$Eph_{k_i} = (EK_i.r_1, EK_i.r_2, EK_i.r_3, EK_i.r_4)$$
   Each auditor/mediator who holds $ek_i$ can compute $ek_i^{-1}.Eph_{k_i}[j] = r_j.G_{Enc}$ and subtract it from the corresponding $CT$ to decrypt.

3. Similarly, sender and receiver each get their own ephemeral keys:
   - Sender gets: $Eph_s = (EK_s.r_1, EK_s.r_3, EK_s.r_4)$ (indices 0, 2, 3 for $r_1, r_3, r_4$; $EK_s.r_2$ is optionally included if senders and receivers are allowed to learn each other)
   - Receiver gets: $Eph_r = (EK_r.r_2, EK_r.r_3, EK_r.r_4)$ (indices 1, 2, 3 for $r_2, r_3, r_4$; $EK_r.r_1$ is optionally included if senders and receivers are allowed to learn each other)

   The `sender_receiver_decryption_needed` flag is true when either `parties_see_each_other` is set OR when there are any encryption keys in the asset (i.e., auditors or mediators with encryption keys).

4. For each mediator affirmation key $MK_j$ (at enc-key index $idx_j$), creator picks a fresh independent random $r_{m_j} \in \mathbb{Z}_p$ and creates:
   $$CT_{m_j} = r_{m_j}.G_{Enc} + MK_j$$
   $$Eph_{m_j} = EK_{idx_j}.r_{m_j}$$
   This allows the mediator (who holds $ek_{idx_j}^{-1}$) to recover $r_{m_j}$ and decrypt $MK_j$. The mediator affirmation key is encrypted separately from the amounts/asset-id.

5. For public encryption keys (auditors who don't need encrypted randomness), create ephemeral keys directly:
   $$Eph_{public\_k_i} = (PK_i.r_1, PK_i.r_2, PK_i.r_3)$$
   where $PK_i$ is the i-th public encryption key. Since these are public, no encryption of $r_j$ is needed.

6. For public mediators (at public enc-key index $idx_j$), creator picks a fresh independent random $r_{public\_m_j} \in \mathbb{Z}_p$ and creates:
   $$CT_{public\_m_j} = r_{public\_m_j}.G_{Enc} + MK_j$$
   $$Eph_{public\_m_j} = PK_{idx_j}.r_{public\_m_j}$$
   Since the encryption key is public, the mediator can recover $r_{public\_m_j}$ directly.

The full leg encryption is:
$$
(CT_s, CT_r, CT_v, CT_{at}, Eph_s, Eph_r, [Eph_{k_0}, Eph_{k_1}, ...], [CT_{m_0}, Eph_{m_0}, CT_{m_1}, Eph_{m_1}, ...],$$
$$[Eph_{public\_k_0}, Eph_{public\_k_1}, ...], [CT_{public\_m_0}, Eph_{public\_m_0}, CT_{public\_m_1}, Eph_{public\_m_1}, ...])
$$

### Decryption
Sender/receiver knows their encryption key pair $(ek, EK)$.

1. Sender computes $r_1 = ek_s^{-1}.Eph_s[0]$ (i.e. $ek_s^{-1}.(EK_s.r_1) = r_1.G_{Enc}$), then checks $CT_s - r_1.G_{Enc} = EK_s$ to confirm it's the right leg.
2. Similarly recover $r_3 = ek_s^{-1}.Eph_s[2]$ and decrypt amount/asset-id by solving discrete log in:
   $$v.H = CT_v - r_3.G_{Enc}$$
   $$at.H = CT_{at} - r_4.G_{Enc}$$
   Since $v$ and $at$ are small (48-bit and 32-bit respectively), discrete log is feasible in reasonable time.

For auditors/mediators with encryption key pair $(ek_i, EK_i)$:
1. Compute $r_j.G_{Enc} = ek_i^{-1}.Eph_{k_i}[j]$ for each $j \in \{1,2,3,4\}$.
2. Decrypt $CT_s, CT_r, CT_v, CT_{at}$ exactly as above.
3. If mediator, additionally recover affirmation key: $r_{m}.G_{Enc} = ek_i^{-1}.Eph_{m}$, then $MK = CT_m - r_m.G_{Enc}$.

## Leg creation proof
The proof for leg creation needs to hide:

- The sender and receiver's identity (or public key)
- The amount being transferred
- The asset being transacted. This is the most complicated part as it needs to prove that all the transaction details are being encrypted for the asset's auditors and mediators without revealing the asset or identity of auditors and mediators.

Recall from the "System model" section that an asset's data is stored in a leaf of the asset curve tree. The leaf now uses two generators $J_0, J_1 \in \mathbb{G}_p$ to distinguish auditor encryption keys from mediator combined keys:
$$
Leaf_{asset\_id} = (P_0.x).\widetilde{G}_0 + (P_1.x).\widetilde{G}_1 + ... + (P_{n+m}.x).\widetilde{G}_{n+m}, \quad Leaf_{asset\_id} \in \mathbb{G}_q
$$
where points are constructed as:
- $P_0 = asset\_id.J_0$
- For each auditor $i$: $P_i = EK_i$
- For each mediator $j$ with enc key index $idx_j$: $P_{n+j} = J_0 + J_1.idx_j + MK_j$

Here $MK_j$ is the mediator's affirmation public key. The $J_1.idx_j$ encodes which auditor encryption key belongs to mediator $j$ (since mediators also have an encryption key they share with auditors, at position $idx_j$ in the auditor key list).

Now this pushes $Leaf_{asset\_id}$ in a different group $\mathbb{G}_q$ (Vesta curve) whereas all public keys are in $\mathbb{G}_p$ (Pallas curve). We use a similar technique used by Curve tree paper. The key idea is that the asset-id and encryption keys for all auditors and mediators is committed in a single Pedersen commitment where the x-coordinate of the item "represents" the item and then we randomize each item and can treat as a group element which can be revealed. Since the $asset\_id \in \mathbb{Z}_p$, we shift it to $\mathbb{G}_p$ by multiplying it by a public $J_0 \in \mathbb{G}_p$. Following describes the general protocol

**Pedersen Commitment to Curve Points**

Commits to elliptic curve points by committing to their x-coordinate the same way curve trees do. Then the knowledge of those committed points can be proven along with generating a re-randomized version of each point.
Given points $P_i \in \mathbb{G}_p$ as 

$$A = [P_0, P_1, ..., P_n]$$ 

Commit to $A$ in a Pedersen commitment of x-coordinate of each $P_i$, i.e. 

$$P_i.x \in C = PedCom(P_0.x, P_1.x, ..., P_n.x) = \sum_{\widetilde{G}_i.P_i.x}$$ 

where $C \in \mathbb{G}_q$ and $P_i.x \in \mathbb{Z}_q$ where $\mathbb{G}_p, \mathbb{G}_q$ are on 2 curves which form a 2-cycle (base field of one equals scalar field of other).

1. Prover randomizes each $P_i$ to get 
  $$A_r = [{P_r}_0, {P_r}_1, ..., {P_r}_n]$$
  
  where 
  
  $$A_r[i] = P_{r_i} = P_i + bl_i.B$$ 

  $$B \in \mathbb{G}_p, \quad bl_i \stackrel{\$}{\leftarrow} \mathbb{Z}_p$$.

2. Prover proves $\forall i, P_i \in \mathbb{G}_p$, i.e. $P_i.x, P_i.y$ are x and y coordinates of a point which lies in group $\mathbb{G}_p$ and $P_i.x, P_i.y \in \mathbb{Z}_q$.
3. Prover proves $\forall i, A_r[i] = A[i] + bl_i.B = P_i + bl_i.B$

The implementation adds a public element $\Delta$ to each $P_i$ as mentioned in the curve tree paper.

In our usage of the above protocol, $A$ corresponds to the points making up the asset leaf:
- $A[0] = asset\_id.J_0$
- $A[i] = EK_i$ for each of the $n$ auditor encryption keys
- $A[n+j] = J_0 + J_1.idx_j + MK_j$ for each of the $m$ mediators

A prover (settlement creator) wants to prove following relations when creating a leg with amount $v$:

1. $CT_v, CT_{at}$ correctly encrypt the amount and asset-id
2. For each auditor with index $i$, $Eph_{k_i} = y.EK_i$ where $EK_i$ is the i-th auditor's encryption key in the asset leaf. For each mediator with index $j$, $Eph_{k_{n+j}} = y.EK_{idx_j}$ (using the enc key at the enc key index for that mediator). This also involves a curve tree membership proof as the creator needs to prove that the asset-id it's proving about is a valid one (in the tree).
3. $v <= MAX\_BALANCE$


**Instance**: $CT_s$, $CT_r$, $CT_v$, $CT_{at}$, $Eph_s$, $Eph_r$, $[Eph_{k_i}]$, $[CT_{m_i}]$, $[Eph_{m_i}]$, $[Eph_{public\_k_i}]$, $[CT_{public\_m_i}]$, $[Eph_{public\_m_i}]$, $Path_r$, $Root$, $G_{Enc}$, $H$, $J_0$, $J_1$, $B$, $B_{blinding}$, curve tree generators

**Witness**: $v, at, r_1, r_2, r_3, r_4, r_1^{-1}, r_2^{-1}, EK_s, EK_r, EK_i, MK_j, [r_{m_j}], [r_{public\_m_j}], [bl_i], Path$.

Above are the high level instance and witness values known before the protocol starts and don't include instance and witness created during protocol execution like the various blindings and $T$ values.


### Prover
1. Prover creates list of points for asset-id and keys as $$A = [at.J_0, EK_1, EK_2, ... EK_n, J_0 + J_1.idx_1 + MK_1, ..., J_0 + J_1.idx_m + MK_m]$$
   where $EK_i$ are auditor encryption keys and $MK_j$ are mediator affirmation keys with their enc key index $idx_j$.
2. Create blindings $blindings = [bl_0, bl_1, ..., bl_{n+m}]$ and randomize $A$ as $$A_r = [AT_r, E_1, ..., E_n, M_1, ..., M_m]$$
   where $AT_r = at.J_0 + bl_0.B_{blinding}$, $E_i = EK_i + bl_i.B_{blinding}$, and $M_j = J_0 + J_1.idx_j + MK_j + bl_{n+j}.B_{blinding}$. $A_r$ is shared with the verifier.
3. Take coordinates of all points in $A$ as $$x=[(AT_r + \Delta).x, (E_1 + \Delta).x, ..., (E_n + \Delta).x, (M_1 + \Delta).x, ..., (M_m + \Delta).x]$$ and similar for $y$.
4. Enforces constraints that for each $i$:
   - $(x_i, y_i)$ is a valid curve point
   - $(x_i, y_i) + bl_i.B_{blinding} = (x_{r_i}, y_{r_i}) + \Delta$ where $(x_{r_i}, y_{r_i}) = A_r[i]$. The RHS is completely public but the LHS involves a fixed point ($B_{blinding}$) scalar multiplication - $bl_i.B_{blinding}$ and a curve point addition - $(x_i, y_i) + bl_i.B_{blinding}$.
5. Since each re-randomized enc-key entry $E_i = EK_i + bl_i.B_{blinding}$, the prover knows $EK_i = E_i - bl_i.B_{blinding}$. For each of the $n$ auditor encryption keys, the ephemeral key relation is $Eph_{k_i}[0] = r_1.EK_i = r_1.(E_i - bl_i.B_{blinding})$.
6. Write $Eph_{k_i}[0] = r_1.E_i + (r_1.bl_i).(-B_{blinding})$. This is a 2-witness Pedersen commitment with witnesses $(r_1, r_1.bl_i)$ and bases $(E_i, -B_{blinding})$. The product $r_1.bl_i$ is enforced by a Bulletproof multiplication constraint. The remaining ephemeral key entries use ratios: $Eph_{k_i}[1] = (r_2/r_1).Eph_{k_i}[0]$, $Eph_{k_i}[2] = (r_3/r_1).Eph_{k_i}[0]$, $Eph_{k_i}[3] = (r_4/r_1).Eph_{k_i}[0]$.
7. For proving $CT_s = r_1.G_{Enc} + r_1^{-1}.Eph_s[0]$ (where $Eph_s[0] = EK_s.r_1$), prover uses witnesses $(r_1, r_1^{-1})$ with bases $(G_{Enc}, Eph_s[0])$. Similarly for $CT_r = r_2.G_{Enc} + r_2^{-1}.Eph_r[1]$ using $(r_2, r_2^{-1})$. The constraints $r_1 \cdot r_1^{-1} = 1$ and $r_2 \cdot r_2^{-1} = 1$ are enforced in Bulletproofs.
8. For proving $AT_r = at.J_0 + bl_0.B_{blinding}$, prover picks random $at_r, bl_{0_r} \in \mathbb{Z}_p$ and creates:
   $$T_{AT_r} = at_r.J_0 + bl_{0_r}.B_{blinding}, \in \mathbb{G}_p$$
9. For the unified Bulletproof commitment, prover commits to the witness array:
   $$witnesses = [v, r_1, r_2, r_3, r_4, r_1^{-1}, r_2^{-1}, r_3/r_1, r_3/r_2, r_4/r_1, r_4/r_2, (r_2/r_1)^*, (r_1/r_2)^*, l_0, l_0 \cdot r_1, ..., l_{n-1}, l_{n-1} \cdot r_1, m_0, m_0 / r_1, ..., m_{m-1}, m_{m-1} / r_1]$$
   where $(r_2/r_1)^*$ and $(r_1/r_2)^*$ are only present if `sender_receiver_decryption_needed` is true. The commitment is:
   $$C_{BP} = b.B_{blinding} + v.G_1 + r_1.G_2 + r_2.G_3 + ... \in \mathbb{G}_p$$
   where $b$ is a fresh random blinding and $G_i$ are Bulletproof generators.
10. Bulletproof constraints enforce:
    - $r_1 \cdot r_1^{-1} = 1$ and $r_2 \cdot r_2^{-1} = 1$
    - $r_1 \cdot (r_3/r_1) = r_3$ and $r_2 \cdot (r_3/r_2) = r_3$
    - $r_1 \cdot (r_4/r_1) = r_4$ and $r_2 \cdot (r_4/r_2) = r_4$
    - If `sender_receiver_decryption_needed`: $r_1 \cdot (r_2/r_1) = r_2$ and $r_2 \cdot (r_1/r_2) = r_1$
    - For each encryption key: $r_1 \cdot l_i = l_i \cdot r_1$
    - For each mediator: $r_1 \cdot (m_i / r_1) = m_i$
    - $v \leq MAX\_BALANCE$ (range proof)
11. **Sender/receiver ciphertext proofs**: Prover picks random $r_{1_r}, r_{1^{-1}_r}, r_{2_r}, r_{2^{-1}_r} \in \mathbb{Z}_p$ and creates:
    $$T_{CT_s} = r_{1_r}.G_{Enc} + r_{1^{-1}_r}.Eph_s[0] \in \mathbb{G}_p$$
    $$T_{CT_r} = r_{2_r}.G_{Enc} + r_{2^{-1}_r}.Eph_r[1] \in \mathbb{G}_p$$

    **Auditor enc-key ephemeral proofs**: For each of the $n$ auditor enc_keys, prover proves $Eph_{k_i}[0] = r_1.E_i + (r_1.bl_i).(-B_{blinding})$. Prover picks random $r_{1_r}, bl_{i \cdot r_1 _r} \in \mathbb{Z}_p$ and creates:
    $$T_{Eph_{k_i}[0]} = r_{1_r}.E_i + bl_{i \cdot r_1 _r}.(-B_{blinding}) \in \mathbb{G}_p$$

    For encryption keys, prover also creates $T_{bl_i} = bl_{i_r}.B$ where $B$ is a different generator, to prove knowledge of $bl_i$ (with commitment $bl_i.B$ provided as instance).

    **Mediator ciphertext and ephemeral proofs**: For each of the $m$ mediator entries in the asset leaf (with re-randomized entry $M_j = J_0 + J_1.idx_j + MK_j + bl_{n+j}.B_{blinding}$), prover proves:
    - $CT_{m_j}$ is correctly encrypted: let $D_j = CT_{m_j} + J_0 + J_1.idx_j - M_j$ (computable from public values). The prover knows witnesses $(r_{m_j}, bl_{n+j})$ in $D_j = r_{m_j}.G_{Enc} + bl_{n+j}.(-B_{blinding})$.  Prover picks random $r_{m_{j,r}}, bl_{(n+j)_r} \in \mathbb{Z}_p$ and creates: $$T_{CT_{m_j}} = r_{m_{j,r}}.G_{Enc} + bl_{(n+j)_r}.(-B_{blinding}) \in \mathbb{G}_p$$
    - $Eph_{m_j}$ is correctly formed: $Eph_{m_j} = r_{m_j}.(E_{idx_j} - bl_{idx_j}.B_{blinding})$ where $E_{idx_j}$ is the re-randomized enc-key at index $idx_j$. Prover proves this using the same $r_{m_j}$ from above and the blinding $bl_{idx_j}$ of the enc-key entry at index $idx_j$. The product $r_{m_j} \cdot bl_{idx_j}$ is enforced by a Bulletproof multiplication constraint (witness $m_j / r_1$ in the BP commitment).

    **Public encryption key proofs**: For each public encryption key $PK_i$, prover picks random $r_{1_r}, r_{2_r}, r_{3_r} \in \mathbb{Z}_p$ and creates:
    $$T_{Eph_{public\_k_i}[0]} = r_{1_r}.PK_i,  \quad T_{Eph_{public\_k_i}[1]} = r_{2_r}.PK_i,  \quad T_{Eph_{public\_k_i}[2]} = r_{3_r}.PK_i$$

    **Public mediator proofs**: For each public mediator $j$ (using public enc key at index $idx_j$), prover proves:
    - $CT_{public\_m_j} - MK_j = r_{public\_m_j}.G_{Enc}$. Prover picks random $r_{public\_m_{j,r}} \in \mathbb{Z}_p$ and creates: $$T_{CT_{public\_m_j}} = r_{public\_m_{j,r}}.G_{Enc}$$
    - $Eph_{public\_m_j} = (r_{public\_m_j} / r_1).Eph_{public\_k_{idx_j}}[0]$. Prover picks random $(r_{public\_m_j} / r_1)_r \in \mathbb{Z}_p$ and creates: $$T_{Eph_{public\_m_j}} = (r_{public\_m_j} / r_1)_r.Eph_{public\_k_{idx_j}}[0]$$
12. For $Eph_{k_i}[1]$, prover picks random $(r_2/r_1)_r \in \mathbb{Z}_p$ and creates:
    $$T_{Eph_{k_i}[1]} = (r_2/r_1)_r.Eph_{k_i}[0] \in \mathbb{G}_p$$
13. For $Eph_{k_i}[2]$, prover picks random $(r_3/r_1)_r \in \mathbb{Z}_p$ and creates:
    $$T_{Eph_{k_i}[2]} = (r_3/r_1)_r.Eph_{k_i}[0] \in \mathbb{G}_p$$
14. For $Eph_{k_i}[3]$, prover picks random $(r_4/r_1)_r \in \mathbb{Z}_p$ and creates:
    $$T_{Eph_{k_i}[3]} = (r_4/r_1)_r.Eph_{k_i}[0] \in \mathbb{G}_p$$
15. For the Bulletproof commitment $C_{BP}$, prover picks random blindings for each witness and creates:
     $$T_{C_{BP}} = b_r.B_{blinding} + v_r.G_1 + r_{1_r}.G_2 + r_{2_r}.G_3 + r_{3_r}.G_4 + r_{4_r}.G_5 + ... \in \mathbb{G}_p$$
     where the blindings correspond to the witness array structure from step 9.
16. For proving $CT_v = r_3.G_{Enc} + v.H$ and $CT_{at} = r_4.G_{Enc} + at.H$, prover picks random $$r_{3_r}, v_r, r_{4_r} \in \mathbb{Z}_p$$ and creates:
    $$T_{CT_v} = r_{3_r}.G_{Enc} + v_r.H \in \mathbb{G}_p$$
    $$T_{CT_{at}} = r_{4_r}.G_{Enc} + at_r.H \in \mathbb{G}_p$$
    Note: $at_r$ was already created in step 8 for the asset-id commitment $T_{AT_r}$.
17. Prover hashes the following to create challenge $c$ as:
    $$c = Hash(A_r, Path_r, Root, Leg\_Enc, C_{BP}, T_{AT_r}, T_{CT_s}, T_{CT_r},$$
    $$T_{Eph_{k_i}[0]}, T_{bl_i}, T_{Eph_{k_i}[1]}, T_{Eph_{k_i}[2]}, T_{Eph_{k_i}[3]}, $$
    $$T_{CT_{m_j}}, T_{Eph_{m_j}}, T_{Eph_{public\_k_i}[j]}, T_{CT_{public\_m_j}}, T_{Eph_{public\_m_j}},$$
    $$T_{C_{BP}}, T_{CT_v}, T_{CT_{at}})$$
    where $Leg\_Enc$ is the full leg encryption structure.
18. Prover creates responses:
    - For $AT_r$ (full Pedersen commitment proof):
      $$resp_{AT} = [at_r + at \cdot c, bl_{0_r} + bl_0 \cdot c]$$
    - For $CT_s$ (partial proof, witnesses from $C_{BP}$):
      $$resp_{CT_s} = [\perp, \perp]$$
      (witnesses $r_1, r_1^{-1}$ are at indices [2], [6] of $resp_{C_{BP}}$)
    - For $CT_r$ (partial proof, witnesses from $C_{BP}$):
      $$resp_{CT_r} = [\perp, \perp]$$
      (witnesses $r_2, r_2^{-1}$ are at indices [3], [7] of $resp_{C_{BP}}$)
    - For $CT_v$ (partial proof, witnesses from $C_{BP}$):
      $$resp_{CT_v} = [\perp, \perp]$$
      (witnesses $r_3, v$ are at indices [4], [1] of $resp_{C_{BP}}$)
    - For $CT_{at}$ (partial proof):
      $$resp_{CT_{at}} = [\perp, \perp]$$
      (witness $r_4$ is at index [5] of $resp_{C_{BP}}$, and $at$ from $resp_{AT}[0]$)
19. For each enc_key $i$, prover creates responses:
    - For $Eph_{k_i}[0]$ (partial proof):
      $$resp_{Eph_{k_i}[0]} = [\perp, bl_{i \cdot r_1 _r} + r_1 \cdot bl_i \cdot c]$$
      (witness $r_1$ is at index [2] of $resp_{C_{BP}}$)
    - For $bl_i$ (partial proof):
      $$resp_{bl_i} = bl_{i_r} + bl_i \cdot c$$
      (proves knowledge of blinding $bl_i$ with commitment $bl_i.B$)
    - For $Eph_{k_i}[1]$ (partial discrete log proof):
      $$resp_{Eph_{k_i}[1]} = (r_2/r_1)_r + (r_2/r_1) \cdot c$$
      (if `sender_receiver_decryption_needed`, witness $r_2/r_1$ is at index [12] of $resp_{C_{BP}}$)
    - For $Eph_{k_i}[2]$ (partial discrete log proof):
      $$resp_{Eph_{k_i}[2]} = (r_3/r_1)_r + (r_3/r_1) \cdot c$$
      (witness $r_3/r_1$ is at index [8] of $resp_{C_{BP}}$)
    - For $Eph_{k_i}[3]$ (partial discrete log proof):
      $$resp_{Eph_{k_i}[3]} = (r_4/r_1)_r + (r_4/r_1) \cdot c$$
      (witness $r_4/r_1$ is at index [10] of $resp_{C_{BP}}$)

    For each mediator $j$, prover creates responses:
    - For $CT_{m_j}$ (Partial2 Pedersen commitment, doesn't store T commitments):
      $$resp_{CT_{m_j}} = [r_{m_{j,r}} + r_{m_j} \cdot c, bl_{(n+j)_r} + bl_{n+j} \cdot c]$$
    - For $Eph_{m_j}$ (partial discrete log proof):
      $$resp_{Eph_{m_j}} = (m_j / r_1)_r + (m_j / r_1) \cdot c$$
      (witness $m_j / r_1$ is at index [14 + 2n + 2j + 1] of $resp_{C_{BP}}$)
20. For public encryption keys and mediators:
    - For each public enc key $i$: $$resp_{Eph_{public\_k_i}} = [r_{1_r} + r_1 \cdot c, r_{2_r} + r_2 \cdot c, r_{3_r} + r_3 \cdot c]$$
      (witnesses $r_1, r_2, r_3$ from $resp_{C_{BP}}$ indices [2], [3], [4])
    - For each public mediator $j$:
      - $resp_{CT_{public\_m_j}} = r_{public\_m_{j,r}} + r_{public\_m_j} \cdot c$ (full discrete log proof)
      - $resp_{Eph_{public\_m_j}} = (r_{public\_m_j} / r_1)_r + (r_{public\_m_j} / r_1) \cdot c$ (partial proof)
21. For relation $C_{BP}$, prover creates response array:
    $$resp_{C_{BP}} = [b_r + b \cdot c, v_r + v \cdot c, r_{1_r} + r_1 \cdot c, r_{2_r} + r_2 \cdot c, r_{3_r} + r_3 \cdot c, r_{4_r} + r_4 \cdot c,$$
    $$r_{1^{-1}_r} + r_1^{-1} \cdot c, r_{2^{-1}_r} + r_2^{-1} \cdot c, (r_3/r_1)_r + (r_3/r_1) \cdot c, (r_3/r_2)_r + (r_3/r_2) \cdot c,$$
    $$(r_4/r_1)_r + (r_4/r_1) \cdot c, (r_4/r_2)_r + (r_4/r_2) \cdot c, ((r_2/r_1)_r + (r_2/r_1) \cdot c)^*, ((r_1/r_2)_r + (r_1/r_2) \cdot c)^*,$$
    $$l_{0_r} + l_0 \cdot c, (l_0 \cdot r_1)_r + (l_0 \cdot r_1) \cdot c, ..., m_{0_r} + m_0 \cdot c, (m_0 / r_1)_r + (m_0 / r_1) \cdot c, ...]$$
    where entries marked with $^*$ are only present if `sender_receiver_decryption_needed` is true. Index [0] is the blinding $b$.
22. The proof is:
    $$(r1cs\_proof, re\_randomized\_path, $$
    $$resp_{CT_s}, resp_{CT_r}, resp_{CT_v}, resp_{CT_{at}}, resp_{AT}, re\_randomized\_points, ped\_comms, $$
    $$C_{BP}, T_{C_{BP}}, resp_{C_{BP}}, $$
    $$[resp_{Eph_{k_i}[0]}, resp_{bl_i}, resp_{Eph_{k_i}[1]}, resp_{Eph_{k_i}[2]}, resp_{Eph_{k_i}[3]}], $$
    $$[resp_{CT_{m_j}}, resp_{Eph_{m_j}}], $$
    $$[resp_{Eph_{public\_k_i}}], [resp_{CT_{public\_m_j}}, resp_{Eph_{public\_m_j}}])$$
    where $r1cs\_proof$ contains the Bulletproof R1CS proof, $re\_randomized\_path$ is the curve tree path proof, $re\_randomized\_points$ are the $A_r$ values, and $ped\_comms$ are the Pedersen commitment proofs for the curve points.

### Verifier

1. Verifier hashes the following to create challenge $c$ as:
   $$c = Hash(A_r, Path_r, Root, Leg\_Enc, C_{BP}, T_{AT_r}, T_{CT_s}, T_{CT_r},$$
   $$T_{Eph_{k_i}[0]}, T_{bl_i}, T_{Eph_{k_i}[1]}, T_{Eph_{k_i}[2]}, T_{Eph_{k_i}[3]}, $$
   $$T_{CT_{m_j}}, T_{Eph_{m_j}}, T_{Eph_{public\_k_i}[j]}, T_{CT_{public\_m_j}}, T_{Eph_{public\_m_j}},$$
   $$T_{C_{BP}}, T_{CT_v}, T_{CT_{at}})$$
2. Verifier enforces the Bulletproof constraints:
   - For each $i$, $(x_i, y_i)$ satisfies the curve equation (curve point validity)
   - For each $i$, $(x_i, y_i) + bl_i.B_{blinding} = (x_{r_i}, y_{r_i}) + \Delta$ (randomization consistency)
   - Curve tree membership: $Path_r$ leads to $Root$
   - Multiplication constraints:
     * $r_1 \cdot r_1^{-1} = 1$ and $r_2 \cdot r_2^{-1} = 1$
     * $r_1 \cdot (r_3/r_1) = r_3$ and $r_2 \cdot (r_3/r_2) = r_3$
     * $r_1 \cdot (r_4/r_1) = r_4$ and $r_2 \cdot (r_4/r_2) = r_4$
     * If `sender_receiver_decryption_needed`: $r_1 \cdot (r_2/r_1) = r_2$ and $r_2 \cdot (r_1/r_2) = r_1$
     * For each encryption key $i$: $r_1 \cdot l_i = l_i \cdot r_1$
     * For each mediator $j$: $r_1 \cdot (m_j / r_1) = m_j$
   - Range proof: $v \leq MAX\_BALANCE$
3. Verifies asset-id commitment $AT_r$ (full Pedersen commitment proof):
   $$resp_{AT}[0].J_0 + resp_{AT}[1].B_{blinding} \stackrel{?}{=} T_{AT_r} + AT_r \cdot c$$
4. Verifies $CT_s$ using shared responses from $resp_{C_{BP}}$:
   $$resp_{C_{BP}}[2].G_{Enc} + resp_{C_{BP}}[6].Eph_s[0] \stackrel{?}{=} T_{CT_s} + CT_s \cdot c$$
   (witnesses $r_1, r_1^{-1}$ at indices [2], [6])
5. Verifies $CT_r$ using shared responses from $resp_{C_{BP}}$:
   $$resp_{C_{BP}}[3].G_{Enc} + resp_{C_{BP}}[7].Eph_r[1] \stackrel{?}{=} T_{CT_r} + CT_r \cdot c$$
   (witnesses $r_2, r_2^{-1}$ at indices [3], [7])
6. Verifies $CT_v$ using shared responses from $resp_{C_{BP}}$:
   $$resp_{C_{BP}}[4].G_{Enc} + resp_{C_{BP}}[1].H \stackrel{?}{=} T_{CT_v} + CT_v \cdot c$$
   (witnesses $r_3, v$ at indices [4], [1])
7. Verifies $CT_{at}$ using shared responses:
   $$resp_{C_{BP}}[5].G_{Enc} + resp_{AT}[0].H \stackrel{?}{=} T_{CT_{at}} + CT_{at} \cdot c$$
   (witness $r_4$ at index [5], $at$ from $resp_{AT}[0]$)

8. For each auditor encryption key $i$, verifies:
   - $Eph_{k_i}[0]$:
     $$resp_{C_{BP}}[2].E_i + resp_{Eph_{k_i}[0]}[1].(-B_{blinding}) \stackrel{?}{=} T_{Eph_{k_i}[0]} + Eph_{k_i}[0] \cdot c$$
     (witness $r_1$ at index [2], product $r_1 \cdot bl_i$ is second response)
   - $bl_i$ commitment:
     $$resp_{bl_i}.B \stackrel{?}{=} T_{bl_i} + (bl_i.B) \cdot c$$
     (proves knowledge of blinding $bl_i$)
   - $Eph_{k_i}[1]$ (if `sender_receiver_decryption_needed`):
     $$resp_{Eph_{k_i}[1]}.Eph_{k_i}[0] \stackrel{?}{=} T_{Eph_{k_i}[1]} + Eph_{k_i}[1] \cdot c$$
     (witness $r_2/r_1$ at index [12] of $resp_{C_{BP}}$)
   - $Eph_{k_i}[2]$:
     $$resp_{Eph_{k_i}[2]}.Eph_{k_i}[0] \stackrel{?}{=} T_{Eph_{k_i}[2]} + Eph_{k_i}[2] \cdot c$$
     (witness $r_3/r_1$ at index [8] of $resp_{C_{BP}}$)
   - $Eph_{k_i}[3]$:
     $$resp_{Eph_{k_i}[3]}.Eph_{k_i}[0] \stackrel{?}{=} T_{Eph_{k_i}[3]} + Eph_{k_i}[3] \cdot c$$
     (witness $r_4/r_1$ at index [10] of $resp_{C_{BP}}$)

9. For each mediator $j$, verifies:
   - $CT_{m_j}$ (Partial2 commitment): Let $D_j = CT_{m_j} + J_0 + J_1 \cdot idx_j - M_j$, then:
     $$resp_{CT_{m_j}}[0].G_{Enc} + resp_{CT_{m_j}}[1].(-B_{blinding}) \stackrel{?}{=} T_{CT_{m_j}} + D_j \cdot c$$
   - $Eph_{m_j}$: Let base $= E_{idx_j}$ (re-randomized enc-key at mediator's enc key index), then:
     $$resp_{Eph_{m_j}}.base \stackrel{?}{=} T_{Eph_{m_j}} + Eph_{m_j} \cdot c$$
     (witness $m_j / r_1$ at index [14 + 2n + 2j + 1] of $resp_{C_{BP}}$)

10. For each public encryption key $i$, verifies:
    - $Eph_{public\_k_i}[0]$: $$resp_{Eph_{public\_k_i}}[0].PK_i \stackrel{?}{=} T_{Eph_{public\_k_i}[0]} + Eph_{public\_k_i}[0] \cdot c$$
    - $Eph_{public\_k_i}[1]$: $$resp_{Eph_{public\_k_i}}[1].PK_i \stackrel{?}{=} T_{Eph_{public\_k_i}[1]} + Eph_{public\_k_i}[1] \cdot c$$
    - $Eph_{public\_k_i}[2]$: $$resp_{Eph_{public\_k_i}}[2].PK_i \stackrel{?}{=} T_{Eph_{public\_k_i}[2]} + Eph_{public\_k_i}[2] \cdot c$$
    (witnesses $r_1, r_2, r_3$ from $resp_{C_{BP}}$ indices [2], [3], [4])

    - $CT_{public\_m_j}$: Let $D_j = CT_{public\_m_j} - MK_j$, then:
      $$resp_{CT_{public\_m_j}}.G_{Enc} \stackrel{?}{=} T_{CT_{public\_m_j}} + D_j \cdot c$$
    - $Eph_{public\_m_j}$: Let base $= Eph_{public\_k_{idx_j}}[0]$, then:
      $$resp_{Eph_{public\_m_j}}.base \stackrel{?}{=} T_{Eph_{public\_m_j}} + Eph_{public\_m_j} \cdot c$$

    $$resp_{C_{BP}}[0].B_{blinding} + resp_{C_{BP}}[1].G_1 + resp_{C_{BP}}[2].G_2 + ... \stackrel{?}{=} T_{C_{BP}} + C_{BP} \cdot c$$
    where the generators $G_i$ correspond to the witness array structure from the prover step 9. This verifies the Schnorr protocol for the Bulletproof commitment.

13. Verifier finally checks the Bulletproofs R1CS proof validates all arithmetic constraints (curve point validity, randomization consistency, curve tree membership, multiplication relations, and range proof).


## Mediator affirmation proof

A mediator for an asset can either accept or reject a leg in a settlement. It needs to prove that it knows the secret keys for the asset's associated mediator key.

Each mediator has two secrets:
- $enc\_sk^{-1}$ - the inverse of its encryption secret key (matching the $EK_{idx}$ in the asset leaf)
- $m\_sk$ - its affirmation secret key (matching the $MK$ in the asset leaf)

The mediator proves knowledge of $enc\_sk^{-1}$ by using it to decrypt $CT_{at}$. It proves $m\_sk$ via a Pedersen commitment $C_{med} = enc\_sk^{-1}.H_0 + m\_sk.H_1$.

Specifically it needs to prove:
1. $enc\_sk^{-1}.Eph_{k_i} = ek\_inv\_part$ where $ek\_inv\_part = CT_{at} - at.H - r_4.G_{Enc} + r_4.G_{Enc} = ?$ (actually: $enc\_sk^{-1}.Eph_{k_i} = r_4.G_{Enc}$ so that $CT_{at} - enc\_sk^{-1}.Eph_{k_i} = at.H$ giving the asset id)
2. $C_{med} = enc\_sk^{-1}.H_0 + m\_sk.H_1$ is correctly formed.

**Instance**: $CT_{at}, Eph_{k_{idx}}, C_{med}, accept/reject, G_{Enc}, H, H_0, H_1$

**Witness**: $enc\_sk^{-1}, m\_sk, at$

Above are the high level instance and witness values known before the protocol starts and don't include instance and witness created during protocol execution like the various blindings and $T$ values.

### Prover

1. Mediator indicates its acceptance or rejection of the leg and this is revealed to the chain and added to transcript.
2. Let $D = Eph_{k_{idx}}$ be the ephemeral key created for this mediator's encryption key in the leg.
3. Prover wants to prove knowledge of $enc\_sk^{-1}$ in:
   $$enc\_sk^{-1}.D = r_4.G_{Enc}$$
   which allows decrypting $CT_{at}$ as $CT_{at} - enc\_sk^{-1}.D = at.H$.
4. Prover also commits to both keys as:
   $$C_{med} = enc\_sk^{-1}.H_0 + m\_sk.H_1, \in \mathbb{G}_p$$
5. Prover picks random $enc\_sk^{-1}_r, m\_sk_r, at_r \in \mathbb{Z}_p$ and creates:
   $$T_{dec} = enc\_sk^{-1}_r.D + at_r.(-H), \in \mathbb{G}_p$$
   $$T_{med} = enc\_sk^{-1}_r.H_0 + m\_sk_r.H_1, \in \mathbb{G}_p$$
6. Prover hashes the following to create challenge $c$ as:
   $$c = Hash(accept/reject, Leg, idx, D, CT_{at}, C_{med}, T_{dec}, T_{med})$$
   where $Leg$ is the leg encryption and $idx$ is the mediator's enc key index in the asset leaf.
7. Prover creates responses:
   $$resp_{enc\_inv} = enc\_sk^{-1}_r + enc\_sk^{-1}.c$$
   $$resp_{m\_sk} = m\_sk_r + m\_sk.c$$
   $$resp_{at} = at_r + at.c$$
8. The proof is
   $$(C_{med}, T_{dec}, T_{med}, resp_{enc\_inv}, resp_{m\_sk}, resp_{at})$$

### Verifier

1. Verifier first checks that position $idx$ corresponds to a mediator in the asset leaf.
2. Verifier adds the mediator's acceptance or rejection to the transcript.
3. Let $D = Eph_{k_{idx}}$ be the ephemeral key at the enc key index for this mediator.
4. Verifier reconstructs the challenge $c$ as:
   $$c = Hash(accept/reject, Leg, idx, D, CT_{at}, C_{med}, T_{dec}, T_{med})$$
5. Verifies the decryption proof:
   $$resp_{enc\_inv}.D + resp_{at}.(-H) \stackrel{?}{=} T_{dec} + CT_{at}.c$$
   $$\implies (enc\_sk^{-1}_r + enc\_sk^{-1}.c).D + (at_r + at.c).(-H) \stackrel{?}{=} enc\_sk^{-1}_r.D + at_r.(-H) + CT_{at}.c$$
6. Verifies the Pedersen commitment to both keys:
   $$resp_{enc\_inv}.H_0 + resp_{m\_sk}.H_1 \stackrel{?}{=} T_{med} + C_{med}.c$$
   $$\implies (enc\_sk^{-1}_r + enc\_sk^{-1}.c).H_0 + (m\_sk_r + m\_sk.c).H_1 \stackrel{?}{=} enc\_sk^{-1}_r.H_0 + m\_sk_r.H_1 + C_{med}.c$$
   Note: the same $resp_{enc\_inv}$ is used in both checks. This proves that the $enc\_sk^{-1}$ used to decrypt the ciphertext is the same one committed to in $C_{med}$ alongside the affirmation key $m\_sk$.
